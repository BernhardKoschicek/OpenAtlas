Feature¶

A feature is the subunit of a place in used in archaeological data recording. E.g. a grave or a building of a site would be considered a feature.

On the index page already entered entities are listed in a Table.

  • Click on the + button to enter a new one.
  • Click on the name of an entry in the list to access the detail view.
  • To edit or delete an entry click on the Edit button in the detail view.

Form fields¶

  • Name
  • Type
  • Case Study
  • Dimension
  • Grave Construction
  • Grave Shape
  • Stylistic Classification
  • Date
  • Description
  • Map

You can edit Case Study, Dimension, Grave Construction, Grave Shape, and Stylistic Classification at Types in the Places tab.

Case Study¶

The respective case study the data was entered into OpenAtlas can be added here, e.g. THANADOS or Morava-Dyje Borderregion.

Dimensions¶

Enter physical dimensions like length, thickness, or degree relevant to the specific feature.

Grave Construction and Grave Shape¶

Hierarchy of Grave Shape and Grave Construction types e.g. Earth pit or Niche grave for construction or Flat grave/Tumuls for shape.

Stylistic Classification¶

Hierarchy of archaeological periods as Prehistory, Protohistory, or Arpadian Age.

Can be linked via tabs to¶

  • Source - when it is referenced there
  • Event - as an event location
  • Reference
  • Stratigraphic unit
  • File
  • Note

Sub units¶

In the OpenAtlas database a feature the subunit of a Place.

A Place can consist of one or more subunits called features (E.g. buildings, graves, pits, ditches, ramparts etc.). Features themselves are structured in the same way as places and can consist of one or more subunits that are labelled as Stratigraphic Unit.

Example: When entering a graveyard, the graveyard as Place is the superior unit. The features or subunits of that place are the graves that the cemetery. Each grave as feature is composed of one or many subunits (stratigraphic units) as well. This could be - for example - the burial of backfilling of the very grave.
